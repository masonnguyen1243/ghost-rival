**[2026-06-19] Update run — validation findings resolved:**

Decision 1 — ghost-dim opacity raised to 40%: `rgba(0,229,255,0.40)`. Rationale: 30% was calibrated on design monitor; 40% maintains visual subordination while ensuring legibility under fluorescent gym lighting.

Decision 2 — Session escape valve: read-only Hall of Fame slide-in panel accessible from within active session (user-initiated, not app-pushed). Pre-session review rejected — doesn't solve the spontaneous mid-session exercise case.

Decision 3 — No Bright Environment Mode for v1. Dark identity preserved. Primary mitigation is ghost-dim opacity fix (Decision 1) + Smart Invert accessibilityIgnoresInvertColors implementation. Note added to spec: "Test in gym lighting before shipping."

Clear fixes applied (no user decision required):

**[2026-06-19] Inherited from PRD §8 — Aesthetic & Tone:**
- Visual: dark-first UI; Ghost data faint/translucent, current-session data bold
- PR Explosion: full-screen, high-contrast, dramatic — felt like a life event; old Ghost visibly retires
- Voice: direct + warm; quiet confidence; genuine (not hyperbolic) celebration
- Ghost copy: self-narrative ("you from last week"), never data-label language
- Anti-references: no cluttered dashboards, no notification spam, no manipulative gamification, no green=good/red=bad body metrics
