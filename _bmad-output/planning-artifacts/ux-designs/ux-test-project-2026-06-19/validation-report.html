Ghost Rival — UX Design Validation Report

ux-test-project-2026-06-19 · 2026-06-19 · 4 reviewers

The spine pair is mechanically sound and ready for architecture on the main implementation path. The token system is clean with zero broken cross-references; all 4 user journeys are present with named protagonists and climax beats; PRD vocabulary is inherited correctly throughout. The rubric walker found no critical issues — Flow coverage, Token completeness, Bloat, Inheritance discipline, and Shape fit are all strong.

The adversarial and accessibility reviewers together surface a single dominant concern: Ghost Rival's core visual mechanic (ghost-dim at 30% opacity) and its dark-only identity create real legibility and accessibility problems in the product's primary use environment — the gym. This is not cosmetic; it undermines the Ghost comparison feature the product is built around. Three critical findings from these reviewers will require spine updates before key-screen mocks or story-dev begins.

The edge-case hunter found 7 critical unhandled cases. Most cause silent data corruption rather than visible crashes: phantom PRs after set deletion, PR loss at force-close, Ghost self-referencing the current session, unit mismatch on draft resume. These need to be captured as explicit behavioral requirements in the architecture phase.

Adversarial Review

review-adversarial.md · 4C / 4H / 4M / 4L

The spec is visually coherent and unusually mature on accessibility and platform deltas, but has critical implementation gaps. The "invisible until it matters" philosophy breaks down in real gym lighting, and the Infinite Goal Engine — the product's key differentiator after Ghost Rival itself — is named but never algorithmically specified.

Critical

ghost-dim 30% opacity unreadable under gym lighting

DESIGN.md Colors §ghost-dim rationale
Calibrated on a design monitor. Under fluorescent gym overheads, low-opacity text loses perceptible contrast faster than the ratio math suggests. The Ghost comparison mechanic becomes invisible precisely when it matters most.
Fix: Test in an actual gym. Likely needs 40–45% opacity. Consider making High Contrast mode the default and the subdued version opt-in.
Critical

No light mode is an operational liability for gym/outdoor use

DESIGN.md Brand & Style
Bright overhead lighting produces glare on dark screens from pupil constriction. No Settings fallback. The brand stance survives a pitch deck but not a parking-lot workout.
Fix: Add "Bright Environment Mode" in Settings — raises surface levels ~30% luminance, preserves all color semantics. Not a light mode; a legibility accommodation.
Critical

Session lockout has no escape valve for mid-session Hall of Fame reference

EXPERIENCE.md Navigation rules
Exercises are added mid-session. Users who add an exercise spontaneously cannot reference past lifts without ending the session. The spec assumes pre-session Ghost review that the flow doesn't facilitate.
Fix: Add read-only Hall of Fame slide-in panel accessible from within session, OR add a "Review Ghosts" step to the session start flow.
Critical

Infinite Goal Engine algorithm entirely unspecified

EXPERIENCE.md PR Explosion Sequence §Step 9
Named and featured throughout both spines. No formula, no input signals, no rounding rules, no edge cases, no display format spec. A developer cannot implement this.
Fix: Add "Infinite Goal Engine" section to EXPERIENCE.md: input signals, calculation logic, rounding, display format ("Next target: 105kg × 4 — est. [date]"), edge cases.
High

Tab bar uses ghost-accent for active state, violating cyan exclusivity rule

DESIGN.md Brand & Style, Components §Tab Bar
DESIGN.md Brand & Style: "Cyan is reserved for one thing: Ghost data." Active tab indicator in cyan directly violates this rule.
Fix: Use ink-primary (white) for active tab state; ink-secondary for inactive.
High

Set pre-fill hierarchy contradicts PRD

EXPERIENCE.md Component Patterns §Set Row; PRD §4.2
PRD: "First Set pre-fills from Ghost data." EXPERIENCE.md: "Pre-fills from previous set within session." Conflict on Set 1. Developer will implement one and be right 50% of the time.
Fix: Define explicit hierarchy: (1) previous Set within session; (2) Ghost data (Set 1); (3) blank (Set 1, no Ghost).
High

Gym Streak mechanic never explained; first reset will feel like a bug

EXPERIENCE.md State Patterns, Home tab
Weekly cadence, reset rules, and Mercy Day interaction are invisible to users. The streak widget shows a number with no context.
Fix: Add tooltip spec for Streak widget first interaction. Add State Pattern for streak reset moment with specific copy.
High

Draft recovery: no "Save as Complete" path

EXPERIENCE.md Session Lifecycle §Phase 5
Users returning hours after a force-close must choose between Resume (reopening) or Discard (losing data). No middle path for "save what was logged."
Fix: Add "Save as Complete" as a third CTA; make it primary when draft is older than 2 hours.

Edge-Case Review

review-edge-cases.md · 57 cases · 7C / 7H / 10M / 4L

57 cases examined across 7 categories. 10 fully handled, 11 partial, 36 gaps. 7 critical unhandled cases — most cause silent data corruption rather than visible crashes. These are engineering requirements, not UX copy issues.

Critical

Swipe-to-delete post-PR within 30s window: phantom Hall of Fame entry, no rollback

EXPERIENCE.md Component Patterns §Set Row; PR Explosion Sequence
Hall of Fame is written at PR detection time. Deleting the triggering set within 30s does not retract the entry. Ghost benchmark is corrupted permanently.
Fix: Spec rollback: if PR-triggering set is deleted within 30s, retract the Hall of Fame entry and restore the previous Ghost.
Critical

PR at exact app force-close: PR silently lost

EXPERIENCE.md Session Lifecycle §Phase 5
Draft recovery restores the session but does not re-run PR detection on recovered sets. The PR is lost with no indication to the user.
Fix: Spec that draft recovery re-evaluates all recovered sets for PR detection and fires retroactive PR Explosion if applicable.
Critical

Stale PR after multi-device sync: phantom PRs accumulate permanently

EXPERIENCE.md Offline-First Behavior §Conflict resolution
PR celebrated on Device A against stale Ghost. After syncing Device B's newer data, the "PR" may not be valid. No re-evaluation or retraction mechanism.
Fix: After sync, re-evaluate Hall of Fame entries against the merged dataset. Flag or retract entries that no longer qualify.
Critical

Ghost referencing in-progress session: "Last Session" self-references current session

EXPERIENCE.md Ghost Comparison Patterns
On first set of a new exercise in the current session, "Last Session" Ghost type would reference that same set, violating the invariant that Ghosts are always from past sessions.
Fix: Ghost candidates exclude the current session_id. "Last Session" = most recently completed session.
Critical

Draft resume after unit change: stored weights in wrong unit

EXPERIENCE.md Session Lifecycle §Phase 5
Draft weights stored in original unit. No conversion spec on resume after unit change.
Fix: Store all weights internally in canonical unit (kg). Unit preference is display-only, never a data transformation.
Critical

0-set session end: undefined Summary, potential 0-value Ghost

EXPERIENCE.md Session Lifecycle §Phase 4
A 0-value Ghost would corrupt all future comparisons for that exercise.
Fix: Block "End Workout" confirmation with 0 sets logged ("No sets logged — are you sure?") or discard 0-set sessions entirely without creating a Ghost.
Critical

SYSTEM_ALERT_WINDOW revoked mid-session: Bubble disappears, no auto-fallback

EXPERIENCE.md Floating Bubble & Live Activity §Android
Notification fallback may not auto-activate on permission revocation. User loses the entire logging UX mid-session.
Fix: On revocation event, immediately promote to persistent notification fallback. Surface one-time toast: "Overlay permission removed — logging via notification instead."

Accessibility Review

review-accessibility.md · 3C / 5H / 6M / 5L

Ghost Rival's accessibility floor is more deliberate than most fitness apps: Reduced Motion, High Contrast Ghost mode, VoiceOver/TalkBack labels, and Dynamic Type reflow are all meaningfully specified. The critical gaps are the cost of two deliberate product decisions (dark-only identity, ghost-dim at 30% opacity) that were never fully reconciled with the accessibility guarantees those decisions undercut.

Critical

Rest timer zero: no persistent signal for deaf-blind users or users in another app

EXPERIENCE.md Interaction Primitives §Haptic; Floating Bubble spec
Haptic + Bubble pulse are instantaneous and non-repeating. WCAG 1.3.3 (Sensory Characteristics). No persistent indicator when user is in another app — exactly the designed use case.
Fix: Repeating haptic (2–3 cycles). Bubble sub-label persists as "Ready" until next set logged. Screen-reader live announcement pushed to foreground app.
Critical

ghost-dim below WCAG AA; High Contrast Ghost not auto-triggered by system Increase Contrast flag

DESIGN.md Colors §ghost-dim; EXPERIENCE.md Accessibility Floor
WCAG 1.4.3. Low-vision users who express accessibility needs via system flags still encounter illegible Ghost data — the core mechanic — with no automatic remedy.
Fix: Read UIAccessibilityIsReduceTransparencyEnabled (iOS) / isHighTextContrastEnabled (Android). Auto-apply High Contrast Ghost when either flag is active.
Critical

No light mode; no Smart Invert compatibility specified

DESIGN.md Brand & Style
WCAG 1.4.3, 1.4.11. Smart Invert without accessibilityIgnoresInvertColors will invert ghost-dim cyan to dark orange-red, destroying the Ghost/PR color semantics.
Fix: Mark Ghost icons, PR Explosion rings, particles, ghost-dim text, ghost-accent indicators, and pr-burst elements with accessibilityIgnoresInvertColors = true on iOS. Document for implementation team.
High

PR Explosion focus management unspecified for screen readers

EXPERIENCE.md PR Explosion Sequence
Fix: Set accessibilityViewIsModal = true. Define traversal order. Return focus to triggering Set Row on dismiss.
High

Ghost Type Selector: selection state not communicated to screen readers

EXPERIENCE.md Component Patterns §Ghost Type Selector
Fix: accessibilityRole="radio", accessibilityState={{ checked: isSelected }}. Full label per option including supplementary Ghost data.
High

30s Set Row edit window insufficient for screen reader users

EXPERIENCE.md Component Patterns §Set Row
Fix: Extend to 90s when VoiceOver / TalkBack is active.
High

Swipe-to-delete Set Row has no screen reader equivalent

EXPERIENCE.md Interaction Primitives
Fix: Implement "Delete" custom action via accessibilityActions (React Native).
Medium

"No ghost yet" placeholder uses ink-disabled (1.5:1) for meaningful informational text

EXPERIENCE.md State Patterns §Ghost Row
Fix: Render in ink-secondary (4.6:1 on surface-raised).
Medium

PR Explosion "Previous ghost" line uses ghost-dim (sub-AA) for comparative data

DESIGN.md Components §pr-explosion-overlay
Fix: On PR Explosion NEW RECORD card specifically, render previous value in ghost-accent at full opacity (≈9.6:1).
